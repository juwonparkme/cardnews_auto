# PPTX 카드뉴스 전환 작업 계획

## 목적

현재 Canva/PDF import 경로는 한글 텍스트와 폰트가 깨질 수 있다.  
다음 단계의 목표는 카드뉴스를 `PPTX` 로 자동 생성하고, 사용자는 `Keynote` 또는 `PowerPoint` 에서 텍스트와 이미지를 직접 수정할 수 있게 만드는 것이다.

## 왜 이 경로로 가는가

- `PDF import` 는 최종 산출물에 가깝고 편집 원본으로 불안정하다
- `Canva import` 는 텍스트 레이어 재해석 과정에서 깨질 수 있다
- `PPTX` 는 텍스트 박스, 이미지 박스, 페이지 구조를 살아 있는 상태로 유지하기 쉽다
- macOS 에서 `Keynote` 로 열어 바로 수정 가능하다

## 최종 목표

1. 카드뉴스 데이터를 기반으로 `PPTX` 파일 자동 생성
2. 디자인은 [`src/inst_cardnews.pdf`](/Users/bagjuwon/Projects/cardnews_auto/src/inst_cardnews.pdf) 와 최대한 동일하게 유지
3. 제목, 아티스트, 타입, summary, 커버 이미지를 각기 수정 가능한 레이어로 출력
4. 사용자는 `Keynote` 또는 `PowerPoint` 에서 직접 수정 가능
5. 필요 시 수정본을 다시 PDF 로 export 가능

## 작업 범위

포함:

- `PPTX` 생성 경로 설계
- 카드뉴스 8페이지 레이아웃 재구성
- 텍스트/이미지 placeholder 분리
- 요약문 길이 budget 재사용
- 이미지 입력 경로 연결
- Keynote 수정용 운영 가이드

제외:

- Canva Brand Template 복구
- Canva live edit 자동화 안정화
- Figma 전용 export 경로

## 단계별 작업 계획

### Phase 1. 포맷/라이브러리 결정

목표:

- TypeScript에서 직접 `pptx` 를 생성할지, 별도 스크립트 경로를 쓸지 결정
- 텍스트 박스, 이미지, 도형, 페이지별 배경 구성 가능 여부 확인

해야 할 일:

1. 후보 라이브러리 조사
2. 한글 텍스트/폰트 처리 가능 여부 확인
3. 이미지 crop/contain/cover 제어 가능 여부 확인
4. 표지/본문/엔드 페이지 레이아웃 표현 가능 여부 확인

완료 기준:

- 선택한 라이브러리 1개 확정
- 간단한 1페이지 `pptx` 샘플 생성 성공

현재 결정:

- 라이브러리: `pptxgenjs`
- 이유:
  - TypeScript/Node 환경에 바로 붙일 수 있음
  - custom layout 지원
  - 텍스트 박스, 도형, 이미지 crop/cover 지원
  - 생성된 `.pptx` 내부에 텍스트 레이어가 유지됨
- Phase 1 샘플 명령:

```bash
pnpm pptx:phase1-sample /tmp/cardnews-phase1-sample.pptx
```

현재 상태:

## 주요 리스크

1. 선택 라이브러리가 한글 폰트/줄바꿈을 기대만큼 재현하지 못할 수 있음
2. `inst_cardnews.pdf` 디자인을 완전히 동일하게 복제하기 어려울 수 있음
3. Keynote 와 PowerPoint 의 렌더 차이가 일부 생길 수 있음
4. 이미지 비율 처리에서 Canva/PDF 와 다른 결과가 나올 수 있음

## 리스크 대응

1. 초기 단계에서 1페이지 샘플로 폰트/줄바꿈 먼저 검증
2. 픽셀 단위 일치보다 `수정 가능성 + 시각적 유사성` 을 우선
3. 텍스트는 폰트 fallback 정책 명시
4. 커버 이미지는 `cover` 기준으로 crop 정책 고정

## 성공 기준

다음을 만족하면 성공으로 본다:

1. 6곡 또는 7곡 기준 `.pptx` 자동 생성
2. Keynote 에서 텍스트 수정 가능
3. 커버 이미지 교체 가능
4. 긴 제목 1개 이상에서도 레이아웃이 크게 무너지지 않음
5. 수정 후 PDF export 결과가 현재 카드뉴스 톤과 크게 다르지 않음

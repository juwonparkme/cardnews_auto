# PPTX 카드뉴스 작업 순서

## 한눈에 보는 흐름

1. 입력 데이터 준비
2. 커버 이미지 준비
3. summary/title/type 정리
4. `PPTX` 생성
5. Keynote 에서 수정
6. 최종 PDF export

## 실제 작업 순서

### 1. 입력 확정

먼저 정한다:

- 카드뉴스 제목
- 곡 순서
- 아티스트명 표기
- 각 곡 타입 `싱글/EP/정규`

이 단계의 결과:

- 카드 순서가 고정된 입력 데이터

### 2. 커버 이미지 준비

입력된 곡 순서에 맞춰 커버 이미지를 준비한다.

기준:

- `images/1.jpg`
- `images/2.jpg`
- `images/3.jpg`
- ...

이 단계의 결과:

- 카드 순서와 정확히 대응하는 이미지 세트

### 3. 텍스트 가공

각 카드마다 아래 값을 정리한다:

- `title`
- `artist`
- `type`
- `summary`
- `imagePath`

규칙:

- 제목은 가능한 한 줄
- 너무 길면 `…` 축약
- summary 는 흰 박스 기준 2~3줄
- 아티스트 표기는 불필요한 괄호 표기 제거 가능

이 단계의 결과:

- `PPTX` 렌더용 카드 데이터

### 4. PPTX 생성

렌더러가 아래 페이지를 만든다:

1. 표지
2. 카드 1
3. 카드 2
4. 카드 3
5. 카드 4
6. 카드 5
7. 카드 6
8. 엔드 페이지

7곡인 경우:

- 카드 페이지 1장 추가

이 단계의 결과:

- 수정 가능한 `.pptx` 파일

현재 구현 명령:

```bash
pnpm pptx:render docs/pptx-cardnews-sample.json /tmp/cardnews-full-sample.pptx
```

샘플 입력:

- [pptx-cardnews-sample.json](./pptx-cardnews-sample.json)

### 5. Keynote 에서 수동 수정

사용자는 여기서 직접 손본다:

- 제목 문구 수정
- summary 다듬기
- 이미지 교체
- 박스 위치 미세 조정
- 표지 문구 수정

핵심:

- 이 단계부터는 Canva import 같은 텍스트 깨짐이 없어야 함
- 수정 가능한 텍스트/이미지 레이어 유지가 목표

### 6. 최종 export

수정이 끝나면:

1. Keynote 에서 PDF export
2. 필요하면 PNG/JPG로도 추가 export
3. 필요 시 그 산출물을 Canva/Figma에 보조 입력으로 사용

## 역할 분리

자동화가 담당:

- 카드 데이터 조합
- 이미지 연결
- 기본 summary 배치
- 페이지 구조 생성
- `.pptx` 파일 생성

사용자가 담당:

- 최종 문구 다듬기
- 세부 위치 미세 수정
- 취향에 따른 강조/삭제
- 최종 export

## 예외 처리 순서

### 긴 제목이 들어오면

1. 한 줄 유지 시도
2. 안 되면 축약
3. 그래도 안 되면 글자 크기 축소

### summary 가 길면

1. 2문장 유지
2. 3줄 기준으로 자르기
3. 정보보다 무드 중심으로 남기기

### 이미지가 어색하면

1. crop 기준 재조정
2. contain 대신 cover 유지 검토
3. 필요 시 수동 교체

## 구현 순서 추천

4. 엔드 페이지 추가
5. 6곡 전체 반복 렌더

- Keynote 에서 열린다

- 이미지 교체가 된다
- PDF export 가 된다
